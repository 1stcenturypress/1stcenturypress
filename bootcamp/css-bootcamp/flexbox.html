CSS 201 - FLEXBOX

Tables were the go-to for page layouts a few years ago. Flexbox should replace your use of tables for anything except an actual table.

The following example shows a parent DIV (with an ID set in CSS) containing (3) child DIV blocks. Because the parent DIV is defined by FLEX-BASIS: 50%;, each child DIV takes up 50%. The third child DIV wraps down due to the FLEX-WRAP: WRAP; in the parent DIV -- otherwise it would just expand the line width to 150%.

Right

NOTE: The entire page has been set to BOX-SIZING: BORDER-BOX in the CSS to allow for the Left and Center to include their border size as part of their total width. (Each DIV would technically be over 50% otherwise, due to their 1px borders.)

You should already see where this structure will allow for easier and more flexible layout options than a table.

Take the following, for example. We create a parent DIV for the DISPLAY and FLEX-WRAP settings with 2 child DIVs. We setup a NAVIGATION class and a CONTENT class respectively, each with their own FLEX-BASIS:, one set at 20%, the other at 80%. And the Header is simply a third child DIV set to FLEX-BASIS: 100% with 0 grow/shrink.

Add a few additional adjustments for colors, fonts, etc. and we have the makings of a decent layout...without tables!
