Adding the BOX Model

Let's say we want to create a 300 x 300 px box...

Box with 5px border and 10px padding (which uses the default CONTENT-BOX settings) actually making it 330px wide

By default, this 300 x 300px box would actually take up more than 300px when we add the border and padding (5px border + 10px padding + 300px interior = 330px)

But with BOX-SIZING we can limit the total px width to the 300px

Box with 5px border and 10px padding PLUS box-sizing set to BORDER-BOX and actually measuring the 300px width

The total interior width is 270px (300px - 5px border - 10px padding)

OUTLINES - the other type of border

Here's a box with the outline defined in blue. It is exterior to all the box elements: border, margin, padding, etc. like a border around the border(s). It has also been given a 5px offset here.

OUTLINES are not calculated in the coding; will not even appear in the inspection box! But it obviously still takes up space on the display.

OVERFLOW For Boxed Content

When the content you assign into a boxed area overflows, you can insert OVERFLOW: and its various options such as HIDDEN, SCROLL, VISIBLE, and AUTO (recommended).
