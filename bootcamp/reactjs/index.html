React.js 101

What is React.js?

On a simplistic level, its a JavaScript library for building user interfaces (UIs). But React.js is actually a way to turn your app or site into reactive/responsive design, which HTML, CSS, and JS can't do.

With React.js, we are back in the world of programming languages. This is not HTML or CSS/Bootstrap, which are basically just display languages. React is more of a hybrid version of JS programming.

Install Node and NPM

We are running NodeJS version 16.14.2 LTS (Long Term Support) and NPM version 8.5.4.

Starting A Project

In Terminal, we take various steps to create a new React.js app.

npm install -g create-new-app does a global install

Then we need a new app in our VS Code explorer:

npx create-react-app my-app which will take a few minutes and perform many actions:

  • installs the new my-app folder in VS Code explorer, with various folders such as node_modules, public, and src plus other files.
  • initializes a git repository
  • installs template dependencies
  • creates a git commit

When finished, it will present a short list of actions you can take, such as:

  • npm start ---- starts the developer server
  • npm run build ---- bundles the app into static files for production
  • npm test ---- starts the test runner
  • npm run eject ---- removes the tool and copies the build dependencies, configuration files and scripts into the app directory (and reminds us that this cannot be undone)

Then it prompts you with a suggested starting point: cd my-app which makes certain you are in the my-app folder directory in the Terminal and npm start which actually starts the app development.

Use both of those suggestions. The first will simply change the directory at your prompt cursor. The second will actually build the React app and automatically launch it in your browser, where you will see a rotating React logo, a reminder to "Edit src/App.js and save to reload", and a link to Learn React from their website.

In the Terminal, you get a listing of the Local (http://localhost:3000) and network (http://192.168.1.100:3000) addresses for the app, and a reminder to create a production build by entering npm run build.

Inside the JSX language

In VS Code we can open files like src/App.js and find that they include commands like IMPORT, function blocks, and even HTML like DIV, HEADER, IMG, P, A, etc. React.js uses its own language, JSX, which is a strange mixture of HTML, CSS, JS, etc. with a few alternate conventions.

For instance, class= becomes className=.

Components

First, in App.js we add import React from 'react'; to the top, which allows us to use React in the page.

Then we can create a component (above the function App() block).

class Item extends React.Component {
render() {
return <h1>Hello React.js World</h1>
} }

And we can invoke it in the body of the function with a simple <Item />, freeing us to change the content in the component-side without touching the function again.

BUT,... we can take it further and house the entire component in its own file (like Classes and Methods are frequently done) and then IMPORT them into the App.js.

If we do this, a few changes must be made. In the new src/MyItem.js (where the component is moved), we add the import React from 'react'; command (remove it from the App.js since it is no longer needed there).

Then below the component block, we must add export default Item; to make it accessible to other files.

Back in the origtinal file (App.js) we add import Item from './MyItem'; and everything should perform as it did before the component was moved.

The Component is now free-standing and can be manipulated as needed without touching the App.js file. Again, this is just like creating separate Class files and invoking them in other files, allowing for single modifications that dynamically update everywhere else.
