jQuery 101 - For Beginners

Why jQuery?

First, jQeuery is a library and not a framework, meaning it is written alongside/within JS rather than necessarily replacing it.

There is an argument that one does not need to learn jQuery if React.js and other frameworks are used instead. However, jQuery can sometimes accomplish in seconds or minutes what a framework like React.js may take much longer to produce through coding.

Furthermore, jQuery is used on millions of websites, making it professionally relevant for well-rounded developers.

React.js is great and efficient for larger coding/customization and therefore should be learned/implemented in larger project needs. jQuery is just easier to "grab and go" on some individual needs.

What is jQuery?

In their own words, jQuery is a JS library designed to implement "HTML document traversal and manipulation, event handling, animation, and Ajax much simpler with an easy-to-use API that works across a multitude of browsers".

How to use jQuery

There are 2 methods for using jQuery: direct download of the components for local implementation or through script insertion from a CDN (content delivery network). Local download will require manual updates by the developer and fresh installs from the user-side as well. Using a CDN has the advantage of latest versions and avoids re-downloading for users who have already acquired jQuery from the same CDN.

We will be using JSDelivr insertion straight into the HTML (near the bottom of the body tag):

script src="https://cdn.jsdelivr.net/npm/jquery@3.6.0/dist/jquery.min.js"

Recommendation:

As a confirmation that all the CDN scripts are actually loaded on your page, consider placing the following function into a SCRIPT at the bottom of the other scripts in your page body:

$(document).ready (function() {
console.log('CDN scripts are loaded!')
})

Original Text

The recommendation just shown is actually a jQuery block reflecting the syntax for common jQuery commands.

$(selector).event(function() {})

If you had an H1 like this:

h1 class="customerClassName" id="customId" with text that says "Original"

You could call either the class or id (or even the entire H1) in a jQuery command:

$("H1").doSomething() ...
$(".customerClassName").doSomething() ...
$("#customId").doSomething() ...

Consider the following, where the text for the H2 tag is being changed as soon as the entire document has been loaded/ready:

$(document).ready(function() {
$("h2.textChange").text("jQuery Syntax");
});

This coding has been applied in this section. Refresh this page and focus on the title for this section ('jQuery Syntax'). You can briefly see that the initial text in the H2 tag is actually "Original Text". With jQuery, it is dynamically changed to "jQuery Syntax" once the document is fully loaded.

SPECIAL NOTE:

All jQuery scripts must be inserted into the HTML page below the jQuery CDN script else it won't function properly.

What about removing/deleting elements?

Technically, performing any .text or .html commands will "override" the original content, which is basically all a .remove or .empty command really does. However, an intentional remove/empty can be accomplished in more than one way.

$("div#parent").remove()
removes the entire parent while
$("div#parent").empty()
empties the parent of its child elements and
const $parent = $("div#parent")
$parent.empty() would perform the same thing.

Just remember that none of these jQuery commands are actually appending, prepending, deleting, emptying, and so forth any HTML code as originally written. The original text, html, etc. are all still there in the code. jQuery merely modifies the content for the browser/front-end manipulation of the page. If you commented out the jQuery commands, the original content would reappear as written.

Adding Event Listeners through jQuery

Events are all user-driven or affected elements. Buttons, form items, etc. The basic syntax for an event listener would be:

$("button").click()

Convert that to a jQuery block like this:

$("button").click(function() { alert("I am clicked!") })

And you end up with the following live example:

.CSS

So far we have used .addClass to insert HTML enhancing code but like .text and .html, we can use .css to directly insert CSS, in at least two ways:

Version 1
$(".box")
.css("background-color", "blue")
.css("border", "10px solid blue")
.css("border-radius", "50%")

Note the slight change in structure, where the key and the value are separate parameters.

Version 2

Can also be coded using objects:

.css({
backgroundColor: "yellow",
border: "10px solid blue",
borderRadius: "10px"
})

Note the change in naming conventions. Objects do not like hyphens so the format switches to camelCase.

.FADE / .SLIDE

Although rarely used, there are .fadeIn() and .fadeOut() events, where an optional time interval can be set as a parameter.

More common, however, are the .slideUp() and .slideDown() events, again with the optional time interval parameter.

Animation

Virtually anything consisting of numbers can be animated in jQuery. Dimensions are the most common example. Colors, however, can not be animated. Computers can't self-calculate the necessary changes between color codes.

Ajax

For our last section of this mini camp, we cover the GET and POST requests for Ajax.

We can grab Star Wars character data from swapi.com in the form of a JSON file by invoking the following Ajax .get():

$.get(`https://swapi.dev/api/people/4`, function(data) {
$("div.swapi").append(`${data.name} | Height: ${data.height} | Mass: ${data.mass}`)
})

Character:

Notice that there is no selector required for a .get command because you are retrieving data from a remote location, usually to be manipulated afterwards, like we did with the DIV above displaying the name, height, and mass of the character.

POST functions in a very smiliar fashion, sending the data to an external location.

$.post(url, data, function(data) {} usually with 2 parameters (key, value).
